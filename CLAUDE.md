```bash
make dev                           # hot reload via air

make test-pkg PKG=internal/store   # run tests for a single package
make lint                          # golangci-lint via go tool

```

## Architecture

**Entry point**: `cmd/server/main.go` — wires Echo, SQLite store, middleware, and routes.

**Request flow**: Echo router → `appmiddleware.Auth` cookie check → handler → `handler.render()` → `html/template` over embedded FS.

**Database layer**:
- `sql/schema/schema.sql` — source-of-truth DDL (also mirrored in `internal/store/migrations.sql` for runtime startup)
- `sql/queries/` — sqlc-annotated SQL; run `make sqlc` after any change
- `internal/db/` — **never edit directly**, fully generated by sqlc
- `internal/store/store.go` — wraps `db.Queries`; `store.New()` opens SQLite and runs `migrate()` at startup

**Handlers** (`internal/handler/`): one file per feature. All use `render(c, "feature/page.html", data)` which clones `layout.html` + parses the page template per request (avoids block conflicts).

**Template rendering**: `render()` in `handler/render.go` uses a `funcMap` with helpers (`divf`, `list`, `dict`). Templates live in `web/templates/`. Login page defines its own full `layout` block (no nav bar).

**Auth**: `internal/middleware/auth.go` checks `auth_token` cookie against `AUTH_TOKEN` env var. `/login` is exempt.

## Domain

- **Accounts**: name, type (`cash | investment | retirement | debt | other`), currency
- **Transactions**: amount stored in **cents** (integer); positive = credit, negative = debit; `date` is `time.Time` parsed from `2006-01-02` form input
- `GetAccountBalance` returns `interface{}` (sqlc limitation with `COALESCE`); use `toInt64()` in `internal/handler/helpers.go` to convert

## Environment

Copy `.env.example` → `.env`. Required vars: `AUTH_TOKEN`. Optional: `DATABASE_URL` (default: `rustyfinancial.db`), `PORT` (default: `8080`).
